# Email Extractor — 项目规划文档

## 一、项目目标

构建一个 Web 应用，用户输入若干网站 URL，系统自动爬取每个网站的页面内容，提取所有电子邮件地址，并通过评分算法判断哪个邮箱最可能是该网站的"官方联系邮箱"，最终统一输出结果。

---

## 二、功能需求

| 功能 | 描述 |
|------|------|
| 批量输入 URL | 用户在前端粘贴/输入多个网址（每行一个） |
| 多页面爬取 | 除主页外，自动尝试 `/contact`、`/about`、`/contact-us` 等常见子页面 |
| 邮箱提取 | 使用正则从 HTML 文本及 `mailto:` 链接中提取所有邮箱 |
| 官方邮箱判定 | 多维度评分：域名匹配 / 常见前缀 / 来源页面权重 / 出现频率 |
| 实时进度反馈 | 前端通过 Server-Sent Events (SSE) 实时展示每个站点的处理状态 |
| 结果展示 | 每个网站展示：官方邮箱（高亮）+ 所有发现的邮箱列表 |
| 一键导出 | 将结果导出为 CSV 文件 |

---

## 三、技术栈

### 后端
- **Python 3.10+**
- **Flask** — 轻量 Web 框架，处理 API 路由 & SSE 推送
- **requests** — HTTP 请求，获取网页内容
- **BeautifulSoup4** — HTML 解析
- **lxml** — HTML 解析加速器
- **tldextract** — 精准提取域名（处理子域名、国家域名等）

### 前端
- 纯 **HTML5 + CSS3 + Vanilla JS**（无框架，零依赖）
- 使用 **EventSource API** 接收 SSE 实时推送
- 响应式布局，支持桌面/移动端

## 五、核心算法：官方邮箱评分

对每个提取到的邮箱计算综合分数，分数最高者判定为官方邮箱：

| 评分维度 | 规则 | 分值 |
|---------|------|------|
| **域名匹配** | 邮箱域名与目标网站主域名一致 | +50 |
| **常见官方前缀** | `info` / `contact` / `admin` / `hello` / `support` / `office` | +20 |
| **来源页面权重** | 来自 `/contact`、`/about` 页面 | +15 |
| **出现频率** | 在多个页面均出现 | +10 × 出现页面数 |
| **非个人邮箱惩罚** | 域名为 Gmail/QQ/163 等免费邮箱 | −30 |
| **格式有效性** | 基本格式校验 | 不通过则丢弃 |

## 六、数据流

```
用户输入 URL 列表
       │
       ▼
  POST /extract
       │
       ▼
  SSE /stream/<task_id>  ←─────────────────────────────┐
       │                                                 │
  for each URL:                                          │
    ├─ 爬取主页 + 常见子页面                              │
    ├─ 提取所有邮箱（正则 + mailto）                       │
    ├─ 对每个邮箱打分                                     │
    ├─ 排序，最高分 = 官方邮箱                             │
    └─ 推送该 URL 的结果 ──────────────────────────────►─┘
       │
       ▼
  前端实时渲染每条结果
       │
       ▼
  用户点击"导出 CSV"
```

---

## 七、反爬处理策略

- 设置随机 `User-Agent` 请求头，模拟浏览器
- 请求超时设置为 10 秒，避免卡死
- 失败时友好提示，不中断整体任务
- 尊重 robots.txt（可选，默认不强制遵守，以提取公开联系信息为目的）

---

## 八、开发步骤

1. `requirements.txt` — 定义依赖版本
2. `extractor.py` — 实现爬取 + 提取 + 评分逻辑（纯函数，方便单独测试）
3. `app.py` — Flask 路由：`GET /`、`POST /extract`、`GET /stream/<task_id>`
4. `templates/index.html` — 前端页面：URL 输入框、进度展示、结果卡片、CSV 导出
